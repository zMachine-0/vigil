1. Overview

This Privacy Policy explains how Vigil ("the Bot", "we", "us") collects, stores, uses, and protects data when used within Discord servers.

By adding or using Vigil, you agree to this Privacy Policy and the Terms of Service.

2. Data We Collect

  • Discord Server IDs and server name (stored in backups)
  • Channel IDs and Role IDs
  • User IDs (for whitelist, quarantine, and permission records)
  • Quarantine reasons and timestamps
  • Security event references (counted, not stored as content)
  • Configuration settings (whitelists, thresholds, permissions)
  • Server backup data (roles, channels, emojis, stickers, server name)

Vigil does not collect message content, direct messages, email addresses, passwords, IP addresses, or payment information.

3. Verification Codes

When a member clicks Verify, a temporary 4-digit code is held in memory only ‚Äî never written to disk ‚Äî for a maximum of 5 minutes. It is deleted immediately after use or expiry.

4. How Data Is Used

  • Detecting raids and destructive activity
  • Performing automated moderation actions
  • Maintaining and restoring server backups
  • Providing security scoring and activity logs
  • Managing the permission hierarchy

Data is never used for advertising, profiling, analytics, or any purpose beyond core bot operation.

5. Automated Decision-Making

Vigil uses automated systems to detect suspicious behaviour and may automatically quarantine, ban, or restrict users. False positives can occur. Server administrators are responsible for reviewing and correcting automated actions.

6. Data Storage

All data is stored locally on the server hosting Vigil as plain-text JSON files. Data is not encrypted by default. The bot operator is responsible for securing the hosting environment and applying encryption at rest if required.

Vigil uses atomic file writes to prevent corruption, sanitizes backup names to prevent path traversal, and auto-recovers corrupted files.

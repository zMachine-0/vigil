Vigil — Privacy Policy

Plain Language Summary: Vigil stores only what it needs to work — server IDs, user IDs for quarantine/whitelist records, and server backup files (roles, channels, emojis, stickers). It never sells your data or shares it with anyone. Verification codes are temporary and vanish within 5 minutes. You can delete your data at any time using bot commands. Use of Vigil also implies agreement to the Terms of Service.

1. Overview

Vigil ("the Bot", "we", "us") is a Discord security bot designed to protect servers from raids, nukes, and unauthorized actions. This Privacy Policy explains what data we collect, how it is used, and your rights regarding that data.

By adding Vigil to your Discord server or using any of its features, you agree to this Privacy Policy and to the Vigil Terms of Service.

2. Data We Collect

Vigil collects and stores only the minimum data required to function. All data is stored locally on the server hosting the bot and is never sold, shared, or transmitted to third parties.

2.1 Server Data (per guild)

DataPurposeStored
Guild IDIdentifies your server across all data filesYes
Guild nameIncluded in backups for identificationYes (in backup files only)
Channel IDsLog channel, verification channel configurationYes
Role IDsOwner role, Trusted Admin role, Whitelist role, Quarantine roleYes

2.2 User Data (per guild)

DataPurposeStored
User IDPermission management, quarantine records, whitelistYes
User tagIncluded in log messagesIn log messages only
Role IDs (quarantined users)Restoring rolesYes, until unquarantined
Quarantine reason & timestampAudit trailYes, until unquarantined

2.3 Server Backups

Backups are stored as plain-text JSON files on the hosting server. They are not encrypted by default and never transmitted externally.

2.4 Verification Codes

Temporary 4-digit codes are stored in memory only for a maximum of 5 minutes and are never written to disk.

2.5 Data We Do NOT Collect

  • Message content
  • Direct messages
  • Passwords or authentication credentials
  • IP addresses or device information
  • Data from users outside servers Vigil is installed in

3. How Data Is Used

  • Operate security features
  • Enforce permission hierarchy
  • Create and restore backups
  • Route logs and alerts
  • Run verification system

4. Data Retention & Deletion

Data TypeRetention

5. Data Security Practices

  • Atomic file writes to prevent corruption
  • Path traversal protection
  • Automatic file recovery

Bot operators are responsible for server security, access control, updates, and encryption if needed.

6. Legal Obligations

Data may be disclosed if legally required by valid court order, subpoena, or government request. Only minimum required data will be disclosed.

7. International Compliance

Users in the EU/UK (GDPR) and California (CCPA) have rights including access, deletion, and correction of data. Vigil does not sell personal data.

8. Your Rights

Server administrators can inspect, modify, or delete stored data. Individual members may request removal through administrators or contact the operator.

9. Backup Restoration Process

Restoring a backup resets server name, deletes current channels/roles, and recreates them from backup. Member lists and message history are not affected.

10. Children's Privacy

Vigil does not knowingly collect data from users under 13. Contact the operator for removal if necessary.

11. Changes to This Policy

Updates may occur at any time. Continued use after updates constitutes acceptance.

12. Contact

Contact the bot operator through the server or platform where Vigil is deployed for questions, deletion requests, or security concerns.

Vigil Setup 101

Complete Command & Permission Reference

Permission Levels

  • True Owner — The actual Discord server owner only
  • Owner Level — True owner + /owner add users + owner role holders
  • Trusted Admin — Owner level + /tadmin add users + TA role holders
  • Everyone — Any server member

Utility

/ping

Shows the bot's latency in milliseconds.

Permission: Everyone

/score

Displays your server’s security score (0–100) with a letter grade (A–F). Includes threat level, quarantined users, log status, backup state, verification gate, and Trusted Admin setup.

Permission: Owner Level

Logging

/log setchannel [channel]

Sets the channel for security alerts and activity logs. Defaults to #vigil-logs if not configured.

Permission: Owner Level

Owner Management

/owner add [user]

Grants owner-level access (excluding whitelist and owner management).

Permission: True Owner

/owner remove [user]

Revokes owner-level access.

Permission: True Owner

/owner list

Displays server owner, owner role (if set), and manually added owners.

Permission: Everyone

/owner setrole [role]

Assigns a role that grants owner-level access automatically.

Permission: True Owner

/owner removerole

Removes owner role assignment.

Permission: True Owner

Trusted Admin Management

/tadmin add [user]

Grants Trusted Admin rank (ban, kick, timeout, quarantine).

Permission: Owner Level

/tadmin remove [user]

Revokes Trusted Admin rank.

Permission: Owner Level

/tadmin list

Shows TA role and manually assigned Trusted Admins.

Permission: Owner Level

/tadmin setrole [role]

Assigns role-based Trusted Admin access.

Permission: Owner Level

/tadmin removerole

Removes Trusted Admin role assignment.

Permission: Owner Level

Whitelist Management

Whitelisted users are fully exempt from automated security actions.

/whitelist add [user]

Permanently exempts a user.

Permission: True Owner

/whitelist remove [user]

Removes a user from whitelist.

Permission: True Owner

/whitelist list

Displays whitelist role and manually whitelisted users.

Permission: True Owner

/whitelist setrole [role]

Assigns role-based whitelist access.

Permission: True Owner

/whitelist removerole

Removes whitelist role assignment.

Permission: True Owner

Quarantine Management

Quarantine strips roles, assigns a restricted role, and restores roles on release.

/quarantine add [user]

Quarantines a user (cannot target whitelisted users).

Permission: Trusted Admin

/quarantine remove [user]

Restores quarantined user roles.

Permission: Trusted Admin

/quarantine list

Lists all quarantined users with timestamps and reasons.

Permission: Trusted Admin

Moderation

All moderation actions require confirmation and are logged.

/ban [user]

Permanently bans a user.

Permission: Trusted Admin

/kick [user]

Kicks a user from the server.

Permission: Trusted Admin

/timeout [user] [duration]

Timeout from 1–40,320 minutes (up to 28 days).

Permission: Trusted Admin

Backup Management

Backups store full server structure including roles, channels, emojis, stickers, and server name.

/backup create [name]

Creates a new backup (confirmation required for overwrite).

Permission: Owner Level

/backup list

Lists all backups with metadata.

Permission: Owner Level

/backup load [name]

This fully restores server structure and cannot be undone.

Permission: Owner Level

Verification

/setupverify [channel] [role]

Creates a button-based 4-digit verification system with expiring codes.

Permission: Owner Level

Auto Backup

Every midnight UTC, Vigil automatically creates an auto-backup:

AUTO_BACKUP-1st-Jan-2025

Only the most recent auto-backup is kept per server.
